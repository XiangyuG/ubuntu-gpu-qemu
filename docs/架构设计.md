# RVT2 加速器架构设计

## 设计目标

在真实硬件到来之前，通过 QEMU 虚拟化搭建一套可完整验证的加速器驱动开发环境。

核心约束：
- 宿主机 Arch Linux x86_64，目标架构 RISC-V 64
- 所有驱动代码在 QEMU 虚拟机内编译和运行
- 设备行为完全由 QEMU 设备模型模拟

## 硬件抽象层

### 计算语义

```
D[m×n] = A[m×k] × B[k×n] + C[m×n]
```

即标准的矩阵 FMA（fused multiply-add），其中 A、B、C 为输入矩阵，D 为输出。当前仅支持 float32 数据类型，矩阵维度上限 4096×4096。

### 硬件流水线（概念）

```
Load → Prefetch → Tile/Pack → 三元矩阵乘核 → 累加/量化 → 写回
```

QEMU 设备模型用软件朴素三重循环模拟此流水线。

### 设备接口

设备通过 PCI 总线挂载，暴露以下接口：

```
+------------------------------------------------+
|             PCI device (0x1234:0x1de2)         |
|             Class: 0x0b40 (Co-processor)       |
+------------------------------------------------+
|  BAR0 [4KiB]  MMIO 寄存器空间                  |
|    +-- 设备标识与状态    (0x00-0x1F)            |
|    +-- 命令队列控制      (0x20-0x3F)            |
|    +-- 完成队列控制      (0x40-0x5F)            |
|    +-- 设备能力          (0x60-0x7F)            |
|    +-- 邮箱通信          (0x80-0x9F)            |
|    +-- HDM 信息          (0xA0-0xAF)            |
|    +-- 固件/故障         (0xB0-0xBF)            |
+------------------------------------------------+
|  BAR2 [1MiB]  HDM 窗口（设备内存）              |
|    +-- CXL Type-2 stub, 主机可直接读写          |
+------------------------------------------------+
|  BAR4 [4KiB]  MSI-X 表                         |
|    +-- Vector 0: 完成中断                       |
|    +-- Vector 1: 故障中断                       |
+------------------------------------------------+
```

## 软件分层

### 第一层：QEMU 设备模型

位置：`qemu/hw/misc/rvt2.c`

职责：
- QOM PCI 设备注册（TypeInfo + class_init + realize）
- BAR0 MMIO 读写处理
- BAR2 HDM 内存窗口（ram_ptr）
- BAR4 MSI-X（2 向量，exclusive bar）
- 描述符队列处理（DMA 读取描述符 → 计算 → DMA 写回）
- 邮箱命令处理（LOAD_FW / INIT / QUERY_CAP / HEARTBEAT）
- 故障处理（队列配置错误、DMA 失败 → STATUS.ERROR + IRQ_FAULT）

### 第二层：内核驱动

#### rvt2_gsp_shim.ko（固件管理模块）

职责：
- `request_firmware()` 加载固件文件
- DMA 上传固件 blob 到设备（LOAD_FW 邮箱命令）
- 邮箱 INIT 握手（QEMU 会在未上传固件时拒绝）
- 能力查询（引擎数、最大描述符大小、固件版本、支持的操作）
- 周期性心跳（5 秒间隔，失败时通过 FAULT_SET 锁存故障）
- 导出 API 供 rvt2_core 使用

#### rvt2_core.ko（主驱动模块）

职责：
- PCI 探测/移除（pcim_enable_device / pci_set_master）
- BAR0 MMIO 映射 + BAR2 HDM 映射
- MSI-X 中断分配和处理
- 调用 rvt2_gsp_attach() 完成固件初始化
- DMA coherent 命令队列和完成队列分配
- misc 字符设备 `/dev/rvt2_0`
- sysfs 属性 `/sys/class/rvt2/rvt2_0/{status,fw_version,engine_count}`
- ioctl 接口：BO_CREATE / BO_INFO / BO_DESTROY / SUBMIT / SUBMIT_RAW / WAIT
- BO 管理：IDR handle 分配、kref 引用计数、DMA coherent 内存、mmap、VMA 追踪
- HDM BO：从 BAR2 bitmap 分配连续页，io_remap_pfn_range 映射
- fence：嵌入 dma_fence、原子 unread_completions 计数、GC 回收

### 第三层：用户态运行时

#### libtmatmulrt.so

简洁的 C API：

```c
int rvt2_open(rvt2_dev_t *dev);
void rvt2_close(rvt2_dev_t *dev);

int rvt2_bo_alloc(rvt2_dev_t *dev, size_t size, uint32_t flags, rvt2_bo_t *bo);
void *rvt2_bo_map(rvt2_dev_t *dev, rvt2_bo_t *bo);
void rvt2_bo_free(rvt2_dev_t *dev, rvt2_bo_t *bo);

int rvt2_submit(rvt2_dev_t *dev, ...);          // 按 BO handle 提交
int rvt2_submit_raw(rvt2_dev_t *dev, ...);       // 提交预构建描述符链
int rvt2_wait(rvt2_dev_t *dev, uint64_t seqno, int64_t timeout_ns);
```

#### compiledd

文本 IR → 二进制描述符翻译器：

```
输入（文本）：ternary_matmul 4 4 4 0 <addr_a> <addr_b> <addr_c> <addr_d>
输出（二进制）：64 字节 struct rvt2_descriptor
```

支持多行 IR 生成多描述符链，可通过 SUBMIT_RAW 一次性提交。

## 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 设备节点 | `/dev/rvt2X` | 简单；稳定后可迁移到 /dev/accel/ |
| BO 模型 | 自定义（非 DRM/GEM） | 避免 DRM 复杂度 |
| 模块拆分 | core + gsp_shim | 固件管理与数据路径分离 |
| 描述符大小 | 64 字节对齐 | 缓存行友好 |
| Fence 模型 | dma_fence + 单调 seqno | 内核标准模式 |
| PCI ID | 0x1234（QEMU vendor） | 避免与真实硬件冲突 |
| 中断路由 | AIA IMSIC | RISC-V virt 机器 MSI-X 所需 |

## 故障处理

```
                  +--------------+
                  |  正常运行    |
                  +------+-------+
                         |
          +--------------+--------------+
          |              |              |
          v              v              v
   DMA 读写失败   队列配置错误   心跳超时/失败
          |              |              |
          v              v              v
   STATUS.ERROR    设备端处理     写 FAULT_SET
   写故障 cpl                          |
   IRQ_FAULT                           v
          |              |      STATUS.ERROR
          +--------------+--------------+
                         |
                         v
                  +--------------+
                  |  故障状态    |
                  | submit->EIO  |
                  | sysfs->ERROR |
                  +------+-------+
                         |
                  CONTROL.RESET
                         |
                         v
                  +--------------+
                  |  重置/恢复   |
                  +--------------+
```
