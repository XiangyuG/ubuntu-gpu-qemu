# RVT2 三元矩阵乘加速器 — 开发指南

## 项目概述

本项目在 QEMU RISC-V 虚拟机中开发一套完整的加速器驱动栈，核心算子为三元矩阵乘加（D = A × B + C）。整个栈从 QEMU 虚拟 PCI 设备模型，到 Linux 内核驱动，再到用户态运行时库，均可在虚拟机中编译、加载和测试，无需真实硬件。

### 架构总览

```
用户态          libtmatmulrt.so / compiledd
                        │ ioctl / mmap / poll
内核驱动        rvt2_core.ko + rvt2_gsp_shim.ko
                        │ MMIO / Doorbell / DMA / Mailbox
固件层          设备管理核心 + 命令处理器（QEMU 内模拟）
硬件层          Load → Prefetch → Tile → 三元矩阵乘核 → 累加/量化 → 写回
```

### 关键技术特性

| 特性 | 实现 |
|------|------|
| PCI 设备 | vendor 0x1234, device 0x1de2, class co-processor |
| 中断 | MSI-X 2 向量（完成 + 故障），需要 AIA IMSIC 支持 |
| DMA | 描述符队列 + 完成队列，coherent DMA |
| 固件 | 通过 LOAD_FW 邮箱命令 DMA 上传，QEMU 验证后 INIT |
| 内存管理 | 自定义 BO（Buffer Object），支持 mmap 到用户态 |
| 同步 | dma_fence + 单调递增 seqno + poll/wait |
| CXL Type-2 | BAR2 1MiB HDM 窗口，驱动端 bitmap 分配器 |

## 环境搭建

### 宿主机要求（Arch Linux）

```bash
# QEMU 和固件
pacman -S qemu-system-riscv
yay -S u-boot-qemu-bin

# 工具
pacman -S sshpass expect cloud-image-utils cdrtools
```

### 镜像准备

```bash
# 下载 Ubuntu RISC-V 服务器镜像
wget https://cdimage.ubuntu.com/releases/24.04/release/ubuntu-24.04.4-preinstalled-server-riscv64.img.xz

# 解压、转换、扩容
xz -dk ubuntu-24.04.4-preinstalled-server-riscv64.img.xz
qemu-img convert -f raw -O qcow2 ubuntu-24.04.4-preinstalled-server-riscv64.img ubuntu-riscv64.qcow2
qemu-img resize ubuntu-riscv64.qcow2 64G

# 生成 cloud-init 种子盘（设置密码）
cloud-localds seed.img user-data
```

### 编译 QEMU（带 rvt2 设备）

```bash
cd qemu/build
../configure --target-list=riscv64-softmmu --enable-slirp
ninja -j$(nproc) qemu-system-riscv64
```

编译完成后，`qemu/build/qemu-system-riscv64` 即为带有 rvt2 加速器设备的 QEMU 二进制。

## 日常开发流程

### 启动虚拟机

```bash
./start-riscv64.sh
# 或后台运行：
nohup ./start-riscv64.sh > vm-console.log 2>&1 &
```

启动脚本会自动检测本地编译的 QEMU（优先使用），并附加 `-device rvt2` 参数。虚拟机参数：
- 4 核 rv64 CPU，8GB 内存
- AIA IMSIC（用于 MSI-X 中断路由）
- SSH 端口转发：宿主机 2222 → 虚拟机 22

### SSH 交互

```bash
./vm-ssh.sh wait              # 等待 SSH 就绪
./vm-ssh.sh setup-key         # 首次：安装 SSH 密钥
./vm-ssh.sh                   # 交互式 shell
./vm-ssh.sh exec "uname -a"   # 远程执行命令
./vm-ssh.sh push src/ /home/ubuntu/src/   # 上传文件
./vm-ssh.sh pull /home/ubuntu/out/ ./     # 下载文件
```

### 编译和加载驱动

```bash
# 宿主机：推送源码
./vm-ssh.sh push driver/ /home/ubuntu/driver/
./vm-ssh.sh push include/ /home/ubuntu/include/

# 虚拟机内：编译（会自动先编译 rvt2_gsp_shim.ko）
cd /home/ubuntu/driver/rvt2_core
make clean all

# 加载双模块（顺序重要！）
sudo mkdir -p /lib/firmware/rvt2
echo -n 'RVT2FW01' | sudo tee /lib/firmware/rvt2/firmware.bin > /dev/null
sudo insmod /home/ubuntu/driver/rvt2_gsp_shim/rvt2_gsp_shim.ko
sudo insmod /home/ubuntu/driver/rvt2_core/rvt2_core.ko

# 验证
ls /dev/rvt2_0
cat /sys/class/rvt2/rvt2_0/status       # 应输出 OK
cat /sys/class/rvt2/rvt2_0/fw_version   # 应输出 0x0100
lspci -k -s 00:01.0                      # 应显示 rvt2_core
```

### 编译和运行测试

## 驱动架构

### 双模块设计

```
rvt2_gsp_shim.ko                    rvt2_core.ko
┌──────────────────┐    EXPORT     ┌─────────────────────────┐
│ rvt2_gsp_attach()├──────────────►│ PCI probe/remove        │
│ rvt2_gsp_detach()│    SYMBOL     │ BAR0 MMIO + BAR2 HDM    │
│ rvt2_gsp_mbox_cmd│    GPL        │ MSI-X IRQ               │
│ rvt2_gsp_latch_  │               │ misc /dev/rvt2_0        │
│     fault()      │               │ sysfs /sys/class/rvt2/  │
│ rvt2_gsp_read()  │               │ BO alloc/mmap/destroy   │
│ rvt2_gsp_write() │               │ submit/wait/fence       │
└──────────────────┘               │ SUBMIT_RAW (chain)      │
                                   └─────────────────────────┘
```

`rvt2_gsp_shim.ko` 必须先于 `rvt2_core.ko` 加载。编译时，core 的 Makefile 会自动先编译 shim 并通过 `KBUILD_EXTRA_SYMBOLS` 引用其符号。

### 固件加载流程

```
Driver                    QEMU Device
  │                           │
  ├─ request_firmware()       │
  ├─ dma_alloc_coherent()     │
  ├─ memcpy(fw blob)          │
  ├─ write FW_ADDR_LO/HI      │
  ├─ write FW_SIZE            │
  ├─ mbox LOAD_FW ──────────► ├─ pci_dma_read(4 bytes)
  │                           ├─ set FW_LOADED
  │   ◄── MBOX_STATUS_DONE ───┤
  ├─ dma_free_coherent()      │
  ├─ mbox INIT  ─────────────►├─ check fw_blob_loaded
  │                           ├─ set READY
  │   ◄── MBOX_STATUS_DONE ───┤
  ├─ mbox QUERY_CAP  ────────►├─ return engine/desc/fw info
  │  ◄── data[0..3]  ─────────┤
  └─ ready = true             │
```

### 命令提交流程

```
Userspace                 Kernel (rvt2_core)           QEMU Device
  │                           │                           │
  ├─ rvt2_submit()            │                           │
  │  ioctl(SUBMIT) ─────────►├─ lookup BOs               │
  │                           ├─ alloc fence (dma_fence)  │
  │                           ├─ write descriptor to cmdq │
  │                           ├─ advance tail             │
  │                           ├─ write DOORBELL ─────────►├─ timer fires
  │                           │                           ├─ fetch descriptor
  │                           │                           ├─ DMA read A,B,C
  │                           │                           ├─ compute D=A×B+C
  │                           │                           ├─ DMA write D
  │                           │                           ├─ write completion
  │                           │  ◄── MSI-X completion ────┤
  │                           ├─ harvest completions      │
  │                           ├─ dma_fence_signal_locked  │
  │  ◄── fence_seqno ────────┤                           │
  │                           │                           │
  ├─ rvt2_wait()              │                           │
  │  ioctl(WAIT) ────────────►├─ wait_event on fence_wq  │
  │  ◄── status=0 (success) ──┤                           │
```

### SUBMIT_RAW 原子链式提交

`RVT2_IOCTL_SUBMIT_RAW` 支持一次提交多个预构建的描述符（例如 compiledd 生成的描述符链），采用全量预验证 + 批量入队的原子语义：

1. **拷贝阶段**：将整个描述符 blob 从用户态拷入内核缓冲区
2. **预分配阶段**：为每个描述符预分配 fence 对象
3. **容量检查**：确认命令队列有足够空间容纳整个链
4. **批量入队**：仅在所有检查通过后，一次性写入所有描述符并触发 doorbell

任何预提交失败都保证 0 个描述符入队。

## 寄存器映射

详见 `docs/registers.md`。主要区域：

| BAR | 大小 | 用途 |
|-----|------|------|
| BAR0 | 4KiB | MMIO 寄存器（控制、状态、队列、邮箱、固件、故障） |
| BAR2 | 1MiB | HDM 窗口（CXL Type-2 设备内存） |
| BAR4 | 4KiB | MSI-X 表（2 个中断向量） |

### 关键寄存器

| 偏移 | 名称 | 说明 |
|------|------|------|
| 0x00 | ID | 设备标识 "RVT2"（0x52565432） |
| 0x08 | STATUS | 设备状态（READY/BUSY/ERROR/FW_LOADED） |
| 0x0C | CONTROL | 设备控制（ENABLE/RESET） |
| 0x34 | DOORBELL | 写触发命令处理 |
| 0x80 | MBOX_CMD | 邮箱命令（NOP/LOAD_FW/INIT/QUERY_CAP/HEARTBEAT） |
| 0xB0-B8 | FW_ADDR/SIZE | 固件上传地址和大小 |
| 0xBC | FAULT_SET | 写 1 锁存 STATUS.ERROR + 触发故障中断 |

## 测试说明

### 测试套件总览

| 测试 | 数量 | 覆盖范围 |
|------|------|----------|
| rvt2_test | 21 | 设备打开/关闭、BO 生命周期、提交/等待、多 fence 回归 |
| rvt2_compiledd_e2e | 9 | 单描述符 + 2 描述符链式提交（经 compiledd 生成） |
| rvt2_cxl_test | 7 | HDM BO 分配/映射/读写/销毁 + 超限分配拒绝 |
| test_compiledd.sh | 5 | compiledd 独立测试（正向 + 负向 IR） |
| rvt2_bench | 4 | 4×4 到 64×64 矩阵乘性能基准 |

### 关键正向测试

- 设备打开/关闭
- BO 分配（4096字节）→ mmap → 读写 → 释放
- 提交 D=A×B+C → 等待完成 → 验证结果与主机参考值一致
- 多次提交验证 seqno 单调递增
- 已信号 fence 的二次等待立即返回
- compiledd 生成描述符 → SUBMIT_RAW 提交 → 结果正确
- 2 描述符链式提交两个独立矩阵乘 → 两个结果都正确

### 关键负向测试

- BO 分配 size=0 返回错误
- 销毁无效 handle 返回 ENOENT
- 已销毁 BO 的 BO_INFO 返回错误
- 提交无效 BO handle 返回 EINVAL
- 等待超时返回 ETIMEDOUT
- HDM BO 超过窗口大小返回 ENOMEM
- compiledd 接收畸形 IR 返回错误

## 未来扩展

- **CXL Type-2 完整实现**：当前 HDM 窗口为简单 RAM stub，未来可扩展为 CXL.cache/CXL.mem 协议实现

- **/dev/accel/ 注册**：当前使用 misc 设备，可迁移到内核加速器子系统
